Название файла содержащего пакет/пакеты с sefdm-символами


Структура пакета:
        |802.11a prmbl|  |OFDM-символы для channel estimation (заголовок)|  |OFDM-символ, содержащий порядковый номер пакета|  |Полезная нагрузка из SEFDM-символов|

  OFDM-символы для channel estimation имеют большую длину по сравнению с SEFDM-символами в полезной нагрузке, но их энергетические спектры идентичны!
  Каждый пакет в файле будет иметь одинаковую полезную нагрузку!


Сткруктура названия файлов:
        sefdm__pckt_[информация_о_пакетах]__hdr_[информация_о_заголовке]__pld_[информация_о_полезной_нагрузке]__sym_[информация_о_sefdm_символе]__.dat
        
  [информация_о_пакетах]            --  1) кол-во пакетов, 2) кол-во нулей между пакетами
  [информация_о_заголовке]          --  1) кол-во ofdm-символов для channel estimation, 2) длина CP (cylic prefix)
  [информация_о_полезной_нагрузке]  --  1) кол-во sefdm-символов в payload, 2) длина CP
  [информация_о_sefdm_символе]      --  1) IFFT size (также соответсвует длине ofdm-символов для channel estimation),
                                        2) длина sefdm-символов, 3) кол-во поднесущих с информацией, 4) дланиа левого GI по частоте, 5) длина правого GI по частоте
                                        6) манипуляция
  

Пример:
        sefdm__pckt_1_0__hdr_2_8__pld_20_6__sym_32_26_20_3_2_bpsk__.dat
Данный файл содержит
  1. 1 пакет, 0 нулей после пакета
  2. 2 ofdm-символа в заголовке с CP равным 8
  3. 20 sefdm-символов в полезной нагрузке с CP 6
  4. При формировании sefdm-символов использовался IFFT на 32 точки;
     длина sefdm-символа 26 из них 20 информационных поднесущих и 3+2 нулевых поднесущих (GI) // также всегда имеется DC-нулевая поднесущая;
     манипулция - bpsk
